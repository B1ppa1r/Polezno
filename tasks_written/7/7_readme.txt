# Тип № 1.

# Основные формулы, используемые в данном типе:

# 1) I = i * x * y, где
# x*y - разрешение, например 128×128
# I - объём информации
# i - количество бит в одном пикселе

# 2) N = 2^i, где
# N - количество символов в алфавите, палитре, наборе
# i - количество бит в одном пикселе


# Пример № 1.

# Какой минимальный объём памяти (в Кбайт) нужно зарезервировать, чтобы можно было сохранить любое растровое изображение размером 512x512 пикселей при условии,
# что в изображении могут использоваться 256 различных цветов?
# В ответе запишите только целое число, единицу измерения писать не нужно.

# Решение:

# 1) Т.к. в изображении могут использоваться 256 различных цветов, то N = 256
# 2) Находим i по формуле N = 2^i. i = 8 бит
# 3) Найдем объем памяти по формуле I = i * x * y.
# I = 8* 512 * 512 = 2097152 бит
# 4) Переведём в Кб. 2097152 бит / 8 = 262144 байт / 1024 = 256 Кб
# Ответ: 256


# Тип № 2.

# Основные формулы, используемые в данном типе:

# 1) V = k * f * i * t, где
# V - объём информации
# k - кол-во дорожек(k = 1 - моно, k = 2 - стерео)
# f - частота дискретизации
# i - глубина кодирования
# t - время звучания

# 2) N = 2^i, где
# N - количество символов в алфавите, палитре, наборе
# i - количество бит в одном пикселе


# Пример № 2.

# Производится четырёхканальная (квадро) звукозапись с частотой дискретизации 32 кГц и 32-битным разрешением.
# Запись длится 3 минуты, её результаты записываются в файл, сжатие данных не производится.
# Определите приблизительно размер полученного файла (в Мбайт). В качестве ответа укажите ближайшее к размеру файла целое число, кратное пяти.

# Решение:

# 1) Подставляем значения, полученные из условия в формулу V = k * f * i * t:
# 32 кГц = 32 000 Гц
# 3 мин = 180 сек
# 2) V = 4 * 32000 * 32 * 180 = 737280000 бит
# 3) Переведём в Мб:
# 737280000 бит / 8 = 92160000 байт / 1024 = 90000 Кб / 1024 = 87,89 Мб
# 4) Ближайшее число, кратное пяти: 90
# Ответ: 90
